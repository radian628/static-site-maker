What the heck is this?

This is a program designed to make creating static sites easier, removing most of the work and flexibility that comes with using raw HTML and CSS. It lets you create sites from Markdown text alone, generating all the boilerplate and organizing your content automatically. It even automatically updates files which index other files (e.g. a blog index that contains every blog post)!

How does it work?

This program uses three major types of files to work: Templates, instances, and workspaces. These are described below.

Templates (.htmltemplate)

Templates define a specific pattern which appears on a bunch of pages. For example, if you want a page with a specific header, footer, and navigation bar, you could use a template to define those elements.

Instances (.htmlinstance)

Let's say you have a template with a header, footer, and navigation bar. It has two places which need content- a title and an article, which obviously vary from page to page, whereas the header, footer, and navigation bar remain constant. While templates define the stuff that remains the same, instances define the stuff that changes from page to page. They're created from an existing template, and content is inserted within them to create a new page.

Workspaces (.ssmworkspace)

Workspaces define an entire site, or some large portion of a site- e.g. a blog. They consist of two pieces: a source folder and a destination folder. The source folder contains the assets (images, audio, video, javascript, etc.) used in your site, along with (this is important) the raw templates and instance files. The destination folder also contains these assets from the source folder, but, rather than containing the templates and instance files, it contains the actual web page (HTML) files generated from them. Here's an analogy: The source is kind of like a blueprint, whereas the destination is the house the blueprint tells you how to build.
